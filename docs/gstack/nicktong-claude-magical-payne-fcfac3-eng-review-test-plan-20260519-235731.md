## Key interactions to verify

- **Brief form submit (happy path)** on `/book/`: fill name + email + company + stage + textarea, click "Send the brief", confirm success card appears in-place and scrolls into view, confirm submission lands in HubSpot CRM as a new contact with all 5 fields mapped.
- **Cal.com inline embed** on `/book/`: scroll to calendar section, confirm cal.com grid loads with brand colour `#1756E8` and light theme, click a slot, confirm cal.com confirmation appears inside the iframe.
- **Form draft restore** on `/book/`: fill 3 fields, refresh page, confirm "Draft restored" note appears with all 3 fields pre-filled, confirm "Clear draft" link works.
- **Form draft expiry**: fill draft, manually set localStorage `savedAt` to >24h ago, refresh, confirm draft silently discarded (no note shown).
- **Consent banner first visit** on `/book/` (or any page): confirm banner appears on first visit only, "Accept" loads GA4 and dismisses banner, "Decline" dismisses banner without loading GA4.
- **Consent banner re-show**: clear `localStorage.ctoondemand_analytics_consent` or set `savedAt` to >12mo ago, refresh, confirm banner re-appears.
- **CTA reroute** on every page: click every visible "Book a Call" link (topbar, header, hero, in-page CTAs, footer), confirm each lands on `/book/`, not `cal.com/nick-tong-cto`.
- **About-strip pill-links** on `/book/`: click "About Nick →" → `/about/`, "The Method →" → `/method/`, "How I Work →" → `/services/`.

## Edge cases

- **HubSpot 4xx response**: temporarily break the form ID in `/js/book-config.js`, submit, confirm error banner shows "We couldn't send this. Check your connection and try again, or email Nick directly." with the form values preserved.
- **Network offline**: DevTools → Network → Offline, submit form, confirm same error banner.
- **Honeypot trip**: DevTools, fill the `_gotcha` field via console (`document.querySelector('input[name="_gotcha"]').value="bot"`), submit, confirm fake-success card shown but nothing lands in HubSpot.
- **Double-click submit**: click Send the brief twice rapidly, confirm only ONE HubSpot request fires (button disabled state holds).
- **Private browsing mode**: open `/book/` in Safari Private / Chrome Incognito, confirm form submission still works, draft doesn't save, consent banner re-shows on every page reload.
- **localStorage quota full**: simulate by filling localStorage to 5MB cap, confirm form still submits and page still works (safeSet returns false silently).
- **Cal.com embed script blocked**: DevTools → Network → block `app.cal.com/embed/embed.js`, confirm fallback link "Calendar didn't load — book directly at cal.com/nick-tong-cto →" appears.
- **GA4 blocked by content-blocker**: install uBlock Origin or similar, accept consent, confirm Vercel Analytics still fires (Network tab shows requests to `vitals.vercel-insights.com`) and the page doesn't break.
- **Photo failed to load**: temporarily break `/images/nick-tong.jpg`, confirm no broken-image icon appears (parent div hidden via `onerror`).
- **47-character names**: fill name field with "Augustinus Bonifacius Hieronymus Adolphus von Tronk" (50 chars), confirm form accepts and HubSpot stores correctly.
- **Reduced motion**: System Preferences → Accessibility → Reduce Motion ON, confirm form-to-success crossfade is disabled and scroll-in animations don't fire.
- **Keyboard navigation**: tab through the entire page, confirm focus rings visible on every interactive element, confirm logical tab order (nav → hero → form fields → submit → "Or grab a slot" link → calendar iframe → about-strip pill-links → footer).

## Critical paths (end-to-end flows that MUST work)

1. **Cold visitor lands on `/book/` from LinkedIn DM → fills brief → submits → HubSpot lead created → success card scrolls into view → email arrives within 1 working day.**
2. **Visitor lands on `/about/` → reads → clicks "Book a Call" CTA → lands on `/book/` → scrolls past form → picks cal.com slot → calendar invite received.**
3. **Returning visitor on `/book/` who refreshed earlier → draft restored → completes remaining fields → submits → lead created.**
4. **GA4 + Vercel Analytics → all 8 funnel events (`page_view`, `book_form_focus_first_field`, `book_form_submit_attempt`, `generate_lead` for both form and calendar, `book_calendar_loaded`, `book_form_submit_error`, `book_cta_click_*`) fire correctly and are visible in both analytics dashboards within 24h.**
5. **Vercel deploy preview URL → DNS cutover → `ctoondemand.co.uk` loads from Vercel with no regressions, no broken assets, no broken nav, no 404s on internal links.**
